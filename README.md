# Hybrid CNN–Transformer Framework for Glaucoma Detection

A deep learning system for automated glaucoma screening from retinal fundus images, combining dual CNN backbones (VGG16 + DenseNet121) with a Transformer encoder for improved local-global feature extraction. Includes Grad-CAM explainability and a Streamlit deployment with LLaMA3-powered clinical explanations.

> **Paper submitted to Springer Nature International Ophthalmology

---

## Results

| Experiment | Dataset | Test Images | Accuracy | AUC | Glaucoma Recall |
|---|---|---|---|---|---|
| Exp 1 | Eye Diseases Classification | 417 | 90.4% | 0.97 | 0.99 |
| Exp 2 | EyePACS + AIROGS-Light V2 | 770 | 94.16% | 0.97 | — |
| Exp 3 | Combined (Focal Loss) | 1,084 | 93.99% | 0.98 | — |

All experiments trained for 50 epochs. Exp 3 uses focal loss to handle class imbalance.

## Architecture

The model runs VGG16 and DenseNet121 **in parallel** on the same input image. Both are pretrained on ImageNet and fine-tuned.

```
Input Image (256×256×3)
    ├── VGG16 (frozen base) → feature map
    └── DenseNet121 (frozen base) → feature map
           ↓
    Concatenate features
           ↓
    Dense layers → reshape to sequence
           ↓
    Multi-Head Self-Attention (8 heads, key_dim=64)
    + Add & LayerNorm + FFN
           ↓
    Classification head → Glaucoma / Normal
```

The Transformer block captures global context (optic disc-to-cup spatial relationships) that CNNs miss due to limited receptive fields.

**Key design choices:**
- Dual CNN branches for richer, complementary feature representations
- Transformer on top of CNN features (not raw patches) — more data-efficient than ViT
- Focal loss in Exp 3 to up-weight hard glaucoma examples
- Grad-CAM applied to both CNN branches independently, then blended

---

## Explainability

Grad-CAM and Grad-CAM++ heatmaps are generated from **both** VGG16 and DenseNet121 branches separately, then blended (weighted by `β`) into a single overlay on the fundus image. Warmer regions = higher activation = model attention.

This matters clinically: the optic disc and cup-to-disc ratio are the ground truth markers for glaucoma. The heatmaps confirm the model is attending to the right structures.

## Deployment

The Streamlit app allows uploading a retinal fundus image and returns:
1. **Prediction** — Glaucoma / Normal with confidence score
2. **Grad-CAM heatmap** — overlaid on the original image
3. **Clinical explanation** — generated by LLaMA3 via Groq API

**Run locally:**
```bash
pip install -r requirements.txt
streamlit run app/app.py
```

Set your Groq API key as an environment variable:
```bash
export GROQ_API_KEY=your_key_here
```

## Limitations

- No patient-level train/test split — images from the same patient may appear across splits
- Threshold calibrated on the test set (not a separate holdout)
- Grad-CAM outputs not yet formally validated by clinicians
- Not tested on external clinical datasets outside the two Kaggle sources

---

## Future Work

- Patient-level splitting to prevent data leakage
- Dedicated threshold calibration set
- External validation on independent clinical data
- Model compression for edge/mobile deployment
- Multimodal input (OCT scans + fundus images)
- Formal clinical validation of Grad-CAM attention regions
